# CLAUDE.md

가라챠토(J-POP 노래방 차트) 리포지토리. Claude가 작업할 때 지키는 규칙.

## 코드 리뷰 피드백 처리 (CodeRabbit 등)

리뷰 코멘트가 오면 아래 순서로 처리한다. 절대 바로 구현부터 하지 않는다.

1. **전부 읽고 판단**: 각 코멘트를 무조건 수용하지 말고, 내 판단으로 수용/반려를 정한다. 필요하면 코드·DB를 실제로 확인해 근거를 잡는다.
2. **파일별 분석 리포트**: 파일별로 묶어, 각 지적이 무슨 문제인지 비전공자도 이해하게 풀어 쓴다. 수용/반려 + 이유 + 심각도(우선순위)를 표시한다.
3. **OK 대기**: 유저의 확인을 기다린다. 이 단계에서 구현하지 않는다.
4. **하나씩 구현**: OK 후 파일별로 하나씩 고치고, 각각 커밋한다.

원안(제안 코드)이 부작용이 있으면 그대로 쓰지 말고 다듬어서 반영하고, 무엇을 왜 바꿨는지 밝힌다.

## 협업 규칙

- 응답·커밋·문서·챗봇 출력·프롬프트에 **em-dash(긴 줄표)를 쓰지 않는다.** 구분이 필요하면 마침표·쉼표·괄호를 쓴다.
- 응답은 한국어.
- 커밋: `type: 한글 요약` + 짧은 `Co-authored-by: Claude <noreply@anthropic.com>`. 이슈 태그·군더더기 없이 "무엇을 했는지" 명확하게.
- 배포: dev가 개발 통합, main이 배포용. 최종 dev→main 머지가 배포.
- DB 변경: 미리보기 → 확인 → 적용 순서로 진행한다.
- 기능 아이디어를 유저가 탐색 중일 때, 분석·추천까지만 하고 "안 함" 결론은 유저가 내리게 한다.
